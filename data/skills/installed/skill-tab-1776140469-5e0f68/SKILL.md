# 영어 → 한국어 번역 + 세 줄 요약

사용자가 제공한 영어 텍스트를 한국어로 번역하고, 번역 결과 상단에 핵심 내용 세 줄 요약을 작성한다.

## 입력 처리

- **텍스트 직접 입력**: 사용자가 채팅창에 붙여넣은 영어 텍스트를 그대로 사용한다.
- **파일 입력**: .txt, .md, .pdf, .docx 등의 파일이 첨부된 경우, 파일 내용을 읽어 텍스트로 추출한 뒤 동일하게 처리한다.
- 두 방식 모두 동일한 번역·요약 파이프라인을 거친다.

## 출력 형식

반드시 아래 순서와 형식을 따른다:

```
## 핵심 요약
• [첫 번째 핵심 내용 — 한국어 한 문장]
• [두 번째 핵심 내용 — 한국어 한 문장]
• [세 번째 핵심 내용 — 한국어 한 문장]

---

## 번역

[번역된 본문 전체]
```

- 요약은 원문의 핵심 메시지를 기반으로 작성한다. 번역문을 축약하는 것이 아니라, 원문 전체를 이해한 뒤 독자에게 가장 중요한 포인트 3가지를 뽑아 한 문장씩 쓴다.
- 번역 본문은 원문의 단락 구조와 줄바꿈을 최대한 유지한다.

## 번역 스타일 판단

텍스트의 성격을 먼저 파악하고, 그에 맞는 번역 스타일을 선택한다.

| 텍스트 유형 | 번역 방향 |
|---|---|
| 기술 문서, 학술 논문, 법률·계약 문서 | 정확한 직역 위주. 원문 구조 유지. 용어 일관성 중시. |
| 뉴스 기사, 블로그, 에세이, 인터뷰 | 자연스러운 의역 위주. 한국어 독자가 읽기 편한 흐름 우선. |
| 혼합형 (예: 기술 블로그) | 기술 용어는 정확하게, 설명 문장은 자연스럽게. |

텍스트 유형이 명확하지 않을 경우 자연스러운 한국어를 우선한다.

## 전문 용어 처리

전문 용어는 문맥과 통용성을 기준으로 아래와 같이 판단한다.

- **번역하는 경우**: 한국어 번역어가 이미 널리 쓰이거나, 번역해도 의미가 명확한 경우. 예) `machine learning` → `머신러닝` 또는 `기계학습`
- **원문을 유지하는 경우**: 영어 원어가 해당 분야에서 더 보편적으로 쓰이는 경우. 예) `API`, `Git`, `CRISPR`, `MVP`
- **병기하는 경우**: 처음 등장하는 중요 전문 용어는 번역어와 영어 원어를 함께 표기한다. 예) `인공지능(Artificial Intelligence)`, `클라우드 컴퓨팅(cloud computing)`
  - 같은 용어가 이후 다시 나오면 병기 없이 번역어(또는 원어)만 사용한다.

## 고유명사·인명·지명·브랜드 처리

- **인명**: 통용되는 한국어 표기가 있으면 사용(예: `Barack Obama` → `버락 오바마`). 없으면 영어 그대로 유지.
- **지명**: 외래어 표기법에 따른 한국어 표기 우선(예: `New York` → `뉴욕`). 표기가 불분명하면 영어 유지.
- **브랜드·제품명·기관명**: 공식 한국어 명칭이 있으면 사용(예: `Apple` → `애플`). 없으면 영어 그대로 유지.
- **논문·책 제목**: 공식 번역 제목이 있으면 사용, 없으면 영어 원제를 유지하고 괄호 안에 직역을 덧붙인다.

## 주의사항

- 요약은 반드시 3문장 정확히 작성한다. 4문장 이상이거나 2문장 이하이면 안 된다.
- 번역 본문을 생략하거나 요약으로 대체하지 않는다. 원문 전체를 번역해야 한다.
- 원문에 오류(문법적 실수, 오탈자 등)가 있어도 임의로 수정하지 않고 의미를 살려 번역한다.
- 번역 과정에서 의미가 불명확한 부분이 있으면 번역 후 [역자 주: ...] 형식으로 짧게 덧붙일 수 있다.
